# LoopLang 專案開發與未來擴充計畫書

## 1. 文件目的
本文件用於整理 LoopLang 專案目前的開發現況、既有架構能力，以及後續可持續擴充的方向與版本路線。  
目的不是一次把所有功能都做完，而是建立清楚、穩定、可執行的演進藍圖，讓專案能從目前的可用版本，逐步成長為具備內容規模、個人化、雲端同步與 AI 輔助能力的語言學習產品。

---

## 2. 專案定位
LoopLang 是一個「極簡、邏輯導向、低操作負擔」的多語會話跟讀工具。  
核心流程圍繞於：

**原文播放 → 停頓 → 翻譯播放 → 停頓 → 下一句**

產品的核心目標不是讓使用者做很多操作，而是讓使用者能在以下情境中，以最低負擔持續練習：

- 通勤
- 駕駛
- 步行移動中
- 旅遊前密集練習
- 碎片化時間反覆跟讀

LoopLang 的本質不是背單字工具、不是文法課程平台、也不是題庫，而是：

- 主題式生活會話播放器
- 多語跟讀工具
- 手機優先的語言學習產品
- 適合長時間、低干擾、近乎 hands-free 的學習體驗

---

## 3. 專案現狀總結（Phase 1）

目前將 LoopLang 的資料封裝進行更嚴謹的區塊化：

*   **雲端同步對象 (`firebase-service.js`)**
    *   `users/{uid}/favorites/{sentenceId}`
    *   `users/{uid}/learned/{sentenceId}` (V2.5 新增進度削去法紀錄)
    *   `users/{uid}/` 根文件目錄 (儲存 `preferences` 及 `recentLearning`)

*   **本機快取對象 (`storage-service.js`)**
    *   `looplang_favorites` 收藏庫
    *   `looplang_learned` 已學會清單
    *   `looplang_recent_learning` 最新進度
    *   `looplang_pref_*` 個人化偏好設定 (含 `filterMode`)

---

### 二、 核心模組梳理

*   **`PlayerService` (播放引擎)**: 採用「動態歌單 (Playlist)」與「全量庫 (BaseSentences)」雙軌制。當接受到不同的 Filtering 狀態時，自動維持與跳轉至正確的 Index，達成流暢但具備進度篩選的學習體驗。結合硬體 `MediaSession` 與 TTS 退化防護。→ 下一句**

並支援以下停頓秒數：
- `0s`
- `1s`
- `2s`
- `3s`

### 3.3 資料模式與同步機制
- 本地 Cache + 雲端同步的 Local-First 雙軌架構
- 整合了 Firebase v10 Auth 進行全站強制登入保護
- 運用 Cloud Firestore 持久化儲存以下資料：
  - 最近學習進度
  - 收藏句子清單
  - 介面顯示偏好
  - 學習節奏設定

### 3.4 硬體與系統整合
- 整合 Wake Lock 防休眠機制
- 整合 Media Session 提供車機 / 耳機控制能力

### 3.5 精緻 UI 基礎 (Premium Glassmorphism)
- 全新極簡留白排版與空氣感漸層背景 (`blob-bg`)
- 採用最高規背景模糊濾鏡，呈現具呼吸感的磨砂玻璃面板
- 泰文排版優化：`line-height: 2.0`，避免上下標音裁切
- 大面積高亮互動與軟陰影，支援無縫平滑過渡

### 3.6 核心亮點與既有能力
- 多頁 HTML + Vue 3 架構
- Firebase Auth 雲端登入與資料推播
- 雙軌音檔切換、Blob 零延遲記憶緩存引擎 (解決行動版裁切) 與跨語系單機智能 TTS 備援
- 底層引擎隔離防護 (`playbackSessionId`) 結合 KeepAlive 心跳波，防禦行動版硬體休眠與多重非同步跳軌
- 字卡隨點互動：獨立於連續播放邏輯，點擊文字單句發音
- 盲聽挑戰模式：觸控解鎖模糊遮罩強迫聽力記憶
- 循環播放與語速 (0.8x - 1.2x) 切換
- Wake Lock 防休眠與 Media Session 音控支援
- 跨網路穩定容錯、增量式音檔本地下載器 (`audioDownloader.html`)

## 4. 現階段架構特性與開發原則

### 4.1 先穩定，再擴大
優先確保：
- 播放流程穩定
- 手機體驗成熟
- JSON 結構一致
- 頁面與模組責任清楚

### 4.2 先內容，再功能
LoopLang 的核心價值仍是「高頻生活會話內容」。  
如果內容不足，再多功能也很難真正提高產品價值。

### 4.3 保持低操作負擔
新增功能不能破壞 LoopLang 的核心哲學：
- 極簡
- 安靜
- 低干擾
- 適合通勤與駕駛
- 盡量接近 hands-free 體驗

### 4.4 架構可替換、可升級
目前以本地 JSON + LocalStorage 為主，未來要能逐步升級為：
- Firebase 同步
- AI 輔助內容生成
- 錄音與發音功能
- 歌詞模式
- PWA / 離線能力
- 更成熟的內容維護流程

## 6. 第一類：內容規模化（Content Scaling）

這是產品價值的核心，目標是讓 LoopLang 從「工具」逐步轉向「具備內容規模的學習平台」。

### 6.1 主題目錄擴充
已於 V2.5 全面建立並梳理了四大語言的 6 大生活與旅遊情境分類框架：
1. **打招呼** (greetings)
2. **餐廳點餐** (dining)
3. **購物與商場** (shopping)
4. **交通與問路** (transport)
5. **飯店住宿** (hotel)
6. **緊急求助** (emergency)

未來的目標是建立自動化流水線，為各分類穩健地擴充單字與句型數量。


### 6.2 難度與分類系統
可在主題與句子層加入：

#### 分級
- `beginner`
- `elementary`
- `intermediate`

#### 分類與標籤
- `category`
- `tags`

例如：
- `#必學`
- `#機場`
- `#餐廳`
- `#緊急`
- `#旅遊`
- `#初學`

用途包括：
- 首頁篩選
- 推薦機制
- 主題排序
- 個人化學習路線

### 6.3 內容治理內部工具
目前內容由本地 JSON 維護，未來建議補 internal tooling：

- JSON 驗證腳本
- 重複 ID 檢查
- `order` 連續性檢查
- `sentenceCount` 校驗
- 音檔路徑有效性檢查
- topic 索引自動生成

這些不是前台功能，但很重要，能避免內容量變大後失控。

## 7. 第二類：播放與學習體驗強化 (✅ 已於 V1.5 達成)

針對跟讀深度、操作靈活性與個人學習效果進行優化。

### 7.1 進階循環模式 (✅)
已支援單句循環播放：`1次`、`3次`、`5次`、`無限`，滿足 shadowing 需求。

### 7.2 播放速度控制 (✅)
已提供：`0.8x`、`1.0x`、`1.2x`。同步於 HTML5 Audio 與 TTS 備援。

### 7.3 多元學習模式引擎 (✅)
除了標準流程，已實作完成：
- **衝刺模式 (Sprint)**：僅播原文，略過翻譯與停頓，達成高壓聽力密集轟炸。
- **挑戰模式 (Challenge/盲聽)**：磨砂遮罩隱藏原始文本，僅顯示翻譯。支援防雷手動解鎖機制。

### 7.5 難句複習系統
提供難句標記功能，例如：
- 不熟
- 常用
- 發音難

之後可自動生成：
- 個人化複習清單
- 近期複習區
- 難句清單

### 7.6 每日練習與推薦
首頁未來可加入：
- 今日推薦 10 句
- 今日主題
- 基於最近學習的推薦
- 基於收藏的推薦

## 8. 第三類：雲端同步與個人化 (✅ 已於 V2.0 達成)

已成功將應用從單機 LocalStorage 升級至 Firebase 雲端雙軌同步。

### 8.1 Auth 閘門系統 (✅)
- 導入 Google Auth SSO，啟動全站路徑把關 (`index.html`, `player.html` 攔截)。

### 8.2 核心資料即時備份 (✅)
- 成功將「最近播放進度 (主題/索引)」、「收藏書籤庫」、「使用者偏好 (語速、停頓、模式)」存入 Firestore `users/{uid}` 架構中。
- 保有離線流暢度，在背景執行 `syncUp` 更新，並於下次登入時自動觸發 `Hydration` 下載覆蓋。

### 8.5 學習統計儀表板
追蹤：
- 每日累計播放句數
- 學習天數
- 收藏總數
- 完成主題數
- 累計學習時間

## 9. 第四類：AI 輔助內容生產（AI Generation）

利用 AI 加速內容產量，定位為 **開發者輔助工具**，而不是一開始就完全自動上線的正式教材來源。

### 9.1 功能定位
未來可新增 **AI 產生句子** 功能，用來加速：
- 新主題建立
- 句型變體延伸
- 多語平行內容生成
- 內容規模化速度

### 9.2 AI 內容草稿生成
輸入：
- 語言
- 主題
- 難度
- 句數
- 語氣（正式 / 日常 / 旅遊）

輸出：
- 原文
- 拼音 / 羅馬拼音 / 粵拼
- 中文翻譯
- 建議 tags
- 符合 JSON 結構的草稿資料

### 9.3 變體句生成器
針對既有句型自動產生變體。

例如：
- 「我要一杯美式」
- 「我要兩杯美式」
- 「我要去冰」
- 「我要外帶」
- 「不要糖」

這特別符合 LoopLang 的句型練習邏輯。

### 9.4 多語平行句同步
針對同一情境，一次產出：
- 日文
- 泰文
- 韓文
- 粵語

這很適合 LoopLang 的多語定位。

### 9.5 AI 主題包生成
輸入情境後，AI 產出完整主題草稿：
- 主題名稱
- 主題簡介
- 句子清單
- 難度
- tags
- 建議句數

### 9.6 個人化 AI 推薦句
當使用者資料足夠後，可依據：
- 常學主題
- 收藏句型
- 最近學習語言
- 練習難度

自動生成個人化推薦句與每日練習內容。

### 9.7 實作原則
AI 功能建議遵循以下原則：

- **AI 生成內容不直接進正式資料**
- 流程應是：  
  **AI 生成 → 人工審核 → 寫入 JSON**
- 優先做內部內容工具，而不是先做給使用者的前台功能
- 輸出格式應對齊 LoopLang 既有 JSON 結構，方便整合

## 10. 第五類：音訊與互動能力升級

### 10.1 錄音跟讀（Echo Mode）
支援使用者錄音並回放，與原音進行手動或 AI 發音比對。

#### 第一階段
- 使用者錄音
- 回放自己的錄音
- 與原始音檔手動比對

#### 第二階段
- 發音比對
- 相似度分析
- 發音提示
- 關鍵詞識別

### 10.2 TTS 能力強化
- 語系指定 voice
- 音色切換
- 語速控制
- 更精細的 fallback 規則

### 10.3 智慧預載策略
根據 Wi-Fi / 行動網路狀態調整音檔預載量，優化流量使用。

可進一步做：
- Wi-Fi 下增加預載量
- 行動網路下節制預載
- 快速切句時調整預載優先順序
- 收藏主題優先快取

### 10.4 Media Session 強化
在車機或耳機控制面板顯示更完整的：
- 翻譯內容
- 學習進度
- 上一句 / 下一句控制
- 暫停恢復狀態保留

## 12. 第七類：平台與離線能力升級

### 12.1 PWA（Progressive Web App）
支援將 LoopLang 安裝至手機主畫面，提供更接近原生 App 的體驗：

- 加入主畫面
- 啟動畫面
- 靜態資源快取
- 更完整的 App 體驗

### 12.2 完整離線模式
逐步支援：

#### 第一階段
- 快取最近主題頁
- 快取最近句子資料

#### 第二階段
- 最近 20 句音檔快取
- 指定主題離線包

#### 第三階段
- 指定主題完整離線使用

### 12.3 內容管理升級

- Firestore content source
- Notion / CMS 匯出 JSON
- Admin Web Panel

但建議等內容量夠大再投入。

### 12.4 開發流程升級
未來可逐步加入：
- 資料檢查腳本
- 音檔檢查腳本
- 自動格式化

## 15. 結論

LoopLang 目前已經有很好的核心骨架：

- 多頁結構
- 穩定播放策略
- 裝置能力整合
- 多語排版適應
- LocalStorage 狀態保存
- 精緻化 UI 與微交互基礎

未來最重要的不是一次加很多功能，而是依照產品核心哲學穩定擴充：

1. 先讓內容更完整  
2. 再讓學習體驗更成熟  
3. 再做同步能力  
4. 再加入 AI 內容生成  
5. 再擴展錄音、歌詞與離線能力  

這樣專案會長得比較穩，也比較不容易失焦。
