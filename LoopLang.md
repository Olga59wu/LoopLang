# LoopLang 專案開發與未來擴充計畫書

## 1. 文件目的
本文件用於整理 LoopLang 專案目前的開發現況、既有架構能力，以及後續可持續擴充的方向與版本路線。
目的不是一次把所有功能都做完，而是建立清楚、穩定、可執行的演進藍圖，讓專案能從目前的可用版本，逐步成長為具備內容規模、個人化、雲端同步與 AI 輔助能力的語言學習產品。

---

## 2. 專案定位
LoopLang 是一個「極簡、邏輯導向、低操作負擔」的多語會話跟讀工具。
核心流程圍繞於：

**原文播放 → 停頓 → 翻譯播放 → 停頓 → 下一句**

產品的核心目標不是讓使用者做很多操作，而是讓使用者能在以下情境中，以最低負擔持續練習：

- 通勤
- 駕駛
- 步行移動中
- 旅遊前密集練習
- 碎片化時間反覆跟讀

LoopLang 的本質不是背單字工具、不是文法課程平台、也不是題庫，而是：

- 主題式生活會話播放器
- 多語跟讀工具
- 手機優先的語言學習產品
- 適合長時間、低干擾、近乎 hands-free 的學習體驗

## 第一部分：現有架構能力 (Current Architecture & Capabilities)

LoopLang 的基底模組與使用者體驗已達穩定狀態，以下為現階段已實作穩定的架構機制：

### 1. 核心資料與同步層 (Local-First Sync)
應用程式採用「本地端為優先，雲端非同步推送」的雙軌架構。保證了無網路時的流暢度與上線時的安全備份。
- **雲端同步對象**：運用 Cloud Firestore 建立對應節點對齊。整合 Firebase v10 Auth 進行全站強制登入保護 (`index.html`, `player.html` 攔截)。
  - `users/{uid}/favorites/{sentenceId}`
  - `users/{uid}/learned/{sentenceId}` 
  - `users/{uid}/` 根目錄 (儲存偏好設定及多語言最新重點進度)
- **本地快取字典**：實作了包含「收藏庫」、「已學會清單」、「個人化偏好設定」等完整的 LocalStorage 映像。
- **配額與節流保護**：具備「雲端寫入防抖閘門（Debounce Sync）」，在使用者連續且快速切換句子時，自動限流暫緩同步，達成大幅降低 Firebase 讀寫費用的商業極限優化。

### 2. 核心播放引擎 (PlayerService)
專為多語情境打造，解決行動端各種刁鑽的播放限制。
- **智能雙軌制**：搭載「動態過濾歌單 (Playlist)」與「全量庫 (BaseSentences)」機制。提供「排已學會」消去法過濾與「僅播標記」考前衝刺邏輯。
- **動態跨庫合集**：支援特殊的「全部內容」模式，於客戶端即時非同步聚合跨主題 JSON 進行無縫連播。
- **防裁切與防休眠引擎**：
  - 導入「Blob 零延遲記憶緩存技術」，將網路串流檔案轉為虛擬本地端，徹底攻克 iOS 瀏覽器常見的前綴裁切與延遲問題。
  - 全面捨棄資源搶佔的 `new Audio()` 惡意預載法，改由底層純 `fetch` 構成快取；同時配置 `playbackSessionId` 執行序隔離防護，搭配實體 Keep-Alive 心跳波阻絕裝置自動休眠，防止非同步跳軌當機。
- **自動降級策略**：遭遇 JSON 中缺乏實體音軌檔案時，自動且穩健地交接至單機智能 Web Speech API (TTS) 備援，支援跨語系智能標點停頓分析。

### 3. 多元播放控制與使用者體驗
針對跟讀深度與個人學習效果進行了高度的開放設定。
- **單句進階循環**：支援切換單句的複誦次數（1次、3次、5次、無限循環）。
- **停頓與語速調整**：可設定跟讀留白停頓秒數（0s~3s），以及微調全局的實際播報速度（0.8x~1.2x）。
- **進階學習模式**：
  - **衝刺模式 (Sprint)**：略過中文與留白停頓，僅播原文，施行高壓聽力密集轟炸。
  - **盲聽挑戰模式**：為預先載入的原文疊加磨砂遮罩屏障，僅顯示翻譯。強制聽力推演，並支援觸控解鎖機制。
- **多語言獨立進度**：支援無縫在四個預設語系切換，自動提取與載入相對應的該語系記憶座標，不產生跨區覆寫衝突。
- **字卡隨點互動**：允許使用者獨立點擊特定字卡發音，不會干涉到背景主引擎的自動播放流程。

### 4. 平台與硬體介接 (Platform & Hardware Integration)
- **PWA 支援 (Progressive Web App)**：支援加入行動裝置立體主畫面。具備獨立啟動畫面與 Service Worker 快取攔截器 (Cache First / Network First 混合策略)，確保資料庫同步零干擾並帶來秒開般的體驗。
- **行動硬體聯動控制**：結合 Media Session API 防制黑畫面時音檔失傳，成功支援實體向車機介面與藍芽耳機上的上一句/下一句硬體多媒體控制操作。

### 5. 精緻化介面 (Premium Glassmorphism)
- 全新極簡留白排版與空氣感漸層背景 (`blob-bg`)。
- 採用高規格背景模糊濾鏡，呈現具呼吸感的磨砂玻璃面板與大面積高亮互動軟陰影。
- **特殊排版解法**：針對泰文與有上下疊加母音特徵的東南亞語系，優化渲染 `Padding` 與 `line-height: 2.0`，確保複雜文字絕不受裁切阻擋。

## 第二部分：未來擴充計畫 (Future Expansion Plan)

本專案現階段的最重要任務不是一次增加過多前台功能，而是依照產品核心哲學由底層逐步穩健擴充。開發順序應秉持「內容治理基建」先於「前台互動功能」的安全準則。

### 優先級 1：內容規模化與基建（Content Scaling & Tooling）
這是產品價值的核心，目標是讓 LoopLang 從單機工具快速擴張為具備資料庫規模的學習平台。
- **主題目錄平滑擴充**：延續現今六大情境（打招呼、點餐、購物等）建立自動化流水線，穩健增加單字與句型豐富度。
- **難度和廣度分類系統**：實作階層化架構（如 Beginner, Intermediate）與細部動態 Hashtag 標籤（ `#必學`、`#機場`）供首頁精細交叉過濾。
- **內容治理內部工具**：建造專用的 JSON 驗證腳本與後台查核系統，用以檢查重複 ID、自動補全缺漏的 `sequence / order` 連續性，核驗內部音檔路徑，防止巨大化資料集引發的前台報錯。

### 優先級 2：AI 輔助內容與平行生成（AI Pipeline）
定位為強大的「內部開發輔助工具」，加速內容生產管線，嚴禁未經審核之 AI 原生資料直上客戶端。
- **草稿與變體句生成**：藉助 Prompt 鏈式管線建立單一情境的變體。例如「我要去冰」延伸至「我要外帶、不要糖」。
- **多語平行句同步引擎**：對相同的劇本情境，快速一鍵橫向翻生出日文、泰文、韓文、粵語的對齊 JSON 草稿檔與中譯。

### 優先級 3：進階音訊與深度互動（Echo Mode）
- **麥克風錄音跟讀 (Echo)**：提供使用者自我糾正回放。進入二期規劃後，可進一步利用 Web Audio API 對本機端進行語音波形包絡分析（Envelope Matching），提供基本的相似發音回饋。
- **TTS 能力進一步客製**：支援音色輪替切換與更細緻的各裝置語音 fallback 客製宣告規則。
- **智慧預載排程**：發展依據裝置網速動態決定的預載策略。判定身處 Wi-Fi 時代替行動網路，加深下一批次音軌檔案預載量。

### 優先級 4：歌詞逐句播放模式 (Lyrics Format Integration)
將目前的會話題材拓展至音樂。
- 支援直接載入對齊時序的歌詞 Json，並完成對應的高亮自動滾動特效。藉由結合音樂節奏，從新的情境面延伸原有的跟讀互動。

### 優先級 5：徹底端對端離線能力（Full Offline Package）
突破目前的 PWA 進度與靜態快取，實施可完全拔除網路的離線資源包。
- 專門開發 ZIP / 雙二進位音檔集合下載模組，將所有的語音檔一併寫入裝置內存，實現在無網路深山或飛行模式下依然能全程流暢使用的主題包全景支援。
